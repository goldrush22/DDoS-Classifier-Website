Evidence & Log Collection Guide

A plain-English guide to the evidence needed to classify denial attacks with D-ACT. PCAPs help identify source and rate conditions; cloud, serverless, and AI logs are needed for sustainability and AI-resource conditions.

PCAP Flow logs Cloud billing Serverless invocation AI usage

Which evidence identifies which attack?

D-ACT can only infer conditions visible in the files supplied. A PCAP alone cannot prove cloud billing, serverless invocation, or AI token consumption.

AttackConditionsMinimum useful evidenceWhy it matters
DoSC0, C1PCAP or flow/request logsShows repeated denial activity from one dominant source.
DDoSC0, C2PCAP or flow/request logsShows repeated denial activity across multiple sources.
LDoSC0, C1, C3PCAP/flow logs with timestampsShows single-source low-rate, periodic, or stealthy degradation.
LDDoSC0, C2, C3PCAP/flow logs with timestampsShows distributed low-rate or periodic degradation.
EDoS_SC0, C1, C4Flow logs + cloud billing or scaling logsLinks one dominant source to cloud autoscaling or chargeable cloud use.
EDoS_DC0, C2, C4Flow logs + cloud billing or scaling logsLinks multiple sources to cloud autoscaling or chargeable cloud use.
DoWC0, C1, C4, C5Flow logs + cloud + serverless invocation logsShows one source triggering cloud-hosted serverless billing exposure.
DDoWC0, C2, C4, C5Flow logs + cloud + serverless invocation logsShows distributed serverless invocation or event-chain billing exposure.
DoAC0, C6AI usage logs, optionally with request logsShows AI token, model-call, tool-call, context, multimodal, or agent activity.

Evidence collection checklist

Network evidence

  • PCAP or flow logs.
  • Source IP/account/client identifiers.
  • Timestamps and request rates.
  • Target hostname, endpoint, port, or route.

Cloud evidence

  • Autoscaling events.
  • Billing and cost exports.
  • Bandwidth, compute, storage, and managed-service usage.
  • Spend alerts and quota changes.

Serverless evidence

  • Function invocation counts.
  • Execution duration and memory allocation.
  • Cold starts and downstream calls.
  • Event-chain or trigger logs.

AI evidence

  • Input and output tokens.
  • Model calls and model class.
  • Context-window size.
  • Tool calls, retrieval calls, multimodal requests, and agent iterations.

Business evidence

  • Hourly online revenue estimate.
  • Customer impact notes.
  • Support tickets and outage times.
  • Unexpected invoice or quota exhaustion evidence.

Preservation notes

  • Preserve raw files before filtering.
  • Record timezone and collection time.
  • Keep hashes or filenames stable.
  • Do not overwrite logs during recovery.
