D-ACT Browser Classifier

A static, client-side web interface for classifying existing or suspected denial-attack artefacts using the C0-C6 Denial Attack Classification Tool. Upload PCAP, flow, cloud, serverless and AI usage evidence, then inspect the inferred conditions, taxonomy classification, and suggested mitigation pathway.

Client-side only No server required PCAP / CSV / JSON / XLSX C0-C6 taxonomy visualisation Mitigation guidance Readiness assessment Incident playbooks

1. Upload evidence files

Files are processed locally in the browser. They are not uploaded to a remote server by this static implementation.

Purpose and assumption: D-ACT is designed to classify existing or suspected denial-attack evidence files. When a file is uploaded, the tool treats it as candidate denial evidence and attempts to classify it against the C0-C6 taxonomy. If the evidence does not satisfy a complete taxonomy rule, D-ACT returns Unclassified rather than forcing an attack label.
Use classic Ethernet/IPv4 .pcap files. If your capture is .pcapng, convert it with editcap -F pcap input.pcapng output.pcap before use.

            Repository notes

            This website is designed for static hosting. Upload index.html, examples.html, simulator.html, mitigations.html, readiness.html, playbook.html, evidence.html, assets/, and optionally examples/ to a public repository. The classifier runs in the browser and mirrors the D-ACT proof-of-concept thresholds used in the paper.

            PCAP-only analysis can infer C0, C1, C2, and C3. Cloud logs are required for C4, serverless logs for C5, and AI usage logs for C6.
