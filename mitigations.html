Denial Attack Mitigation Strategies

A practical guide to mitigation pathways for each D-ACT denial attack classification. Use this page after D-ACT identifies a class, or as a reference for designing controls against the C0-C6 denial taxonomy.

DoS / DDoS Low-rate denial EDoS / DoW DoA and AI-resource exhaustion

How to use this page

Each mitigation card corresponds to one D-ACT classification. The guidance is organised into immediate triage, technical controls, and monitoring or follow-up. These strategies are defensive and should be adapted to the affected environment, baseline traffic, cloud provider, serverless platform, and AI provider.

D-ACT classifies existing or suspected denial-attack evidence. Mitigation recommendations are generated from the identified class and should be treated as triage guidance, not as a substitute for incident-response judgement.

Mitigation catalogue

The catalogue covers the nine primary taxonomy classes used by the D-ACT evaluation corpus, plus the negative-control Unclassified outcome.

Operational note

D-ACT mitigation guidance is intentionally condition-aware. For example, a DDoS result emphasises distributed ingress protection, while DoW and DDoW results add serverless billing controls. DoA results add AI-specific controls for tokens, model calls, context windows, tool use, multimodal generation, and agentic recursion.
