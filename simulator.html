Denial Attack Simulator

A purely visual teaching tool for the D-ACT C0-C6 denial taxonomy. Select an attack class to see how the attacker, victim, packets, cloud/serverless/AI resources, active conditions, and mitigation strategies relate to one another.

How this simulator works

This simulator does not generate traffic, replay PCAP files, or interact with any network. It is an explanatory visual model. The animation represents the theory behind each denial attack class: source structure, rate profile, cloud exposure, serverless billing exposure, and AI-resource consumption.

Scope: visual education only. Use the classifier to classify existing evidence files and the example use cases to run reproducible synthetic artefacts.

Victim financial profile simulator

Optional: enter a business name, ordinary profit figure, and the hourly profit you believe the online service generates while it is available. The simulator uses the hourly online-service value to model lost revenue during availability attacks, and uses interim metered-cost coefficients to model sustainability attacks. These values are editable educational coefficients and can be replaced with empirical research data later.

Sustainability impact model
Model basis: availability-denial classes simulate lost online-service profit caused by outage, lag, or degraded access. Sustainability-denial classes additionally simulate metered cost exposure because cloud, serverless, and AI platforms commonly charge for resources used, such as cloud resources, function duration/invocations, and AI token or tool usage. This simulator does not estimate actual provider invoices or actual business losses. It demonstrates the financial logic of denial attacks using transparent interim coefficients.

      Theory note

      The simulator uses the same nine primary D-ACT taxonomy classes as the evaluation corpus: DoS, DDoS, LDoS, LDDoS, EDoSS, EDoSD, DoW, DDoW, and DoA. Green condition badges indicate the conditions required by the selected class. Red condition badges indicate conditions outside the selected class. This makes the taxonomy tree visible without requiring the user to read formal notation.
